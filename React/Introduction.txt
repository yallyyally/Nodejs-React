[2강] 리액트의 Virtual DOM
지속해서 데이터가 변화하는 대규모 애플리케이션을 구축하기 위함
MVC - 데이터 Model 화면 View 사용자 이벤트 관리 Controller
->MVVM, MVW 파생
공통점: Model
양방향 바인딩: 모델의 값이 변하면 view 에서도 변화, view에서 변하면 model 에서도 변화...
'변화' (Mutation) : 특정 이벤트가 발생햇을때 모델이 변화를 일으키고, 그 변화에 따라 다른 DOM을 가져와서 
어떤 방식으로 VIEW를 업데이트 할지 로직ㅇ (< 프레임워크나 라이브러리에 ㄷ따라 다르다)
페북: 데이터가 바뀌면 뷰르 ㄹ날리고 새로 만들자!
->성능적으로 문제 잇자나 1
 -> Virtual DOM!!!!
 변화가 일어나면 실제 DOM에 넣는게 아니구, 자바스크립트 가상 DOM에 렌더링 하고,
 기존 DOM과 비교해 변화가 ㅍ있는 곳만 렌더링.
